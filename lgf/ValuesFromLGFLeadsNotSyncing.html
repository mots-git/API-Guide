Values Not Syncing

1. If a client is having an issue where they have an API connection to a LGF and only some of the values are syncing would indicate that they have a mapping issue. In the CRM they have to map the different fields to each other, for example if the LGF has the fields below this is how they should be mapped:

LinkedIn CRM
First Name First Name
Last Name Last Name
Country Country
Work Email Work Email

What we see in some cases is only some of the data from the fields above will sync across and this means that the client simply has to fix the mapping in their CRM. For this send across the relevant setup guide from here.

If the field which is not syncing needs a standardization value please send the client the follow sheet here. This contains the values for countries, states, job functions, etc which is needed for them to map correctly.
